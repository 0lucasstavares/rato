# M4 — Workbench Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** tmux+worktree workbench with agent adapters and an R-tiered approval flow — per spec `docs/superpowers/specs/2026-06-11-m4-workbench-design.md` (binding, esp. its Decisions table).

**Conventions (binding, as M3):** ULID ids, Clock-trait time, single-writer store actor, PROTO_VERSION 1 additive, explicit-path commits (NEVER `git add -A`), `cargo test --workspace` + clippy green per task, frontend `npm run check`+`build` green. PATH: `export PATH="$HOME/.cargo/bin:$HOME/.local/bin:$PATH"`.

### Task 1: store migration v4 + repos
Tables per §10 DDL: `approvals` (all columns incl. execution JSON), `actions`, `agent_runs`, `blobs(id PK, sha256 UNIQUE, bytes BLOB, created)`. Indexes `approvals(status, expires_at)`, `agent_runs(status, started)`. Store methods: `insert_approval(NewApproval) -> Approval`, `pending_approvals()`, `get_approval(id)`, `decide_approval(id, status, decided_at, decided_via, note)` (only from `pending`; returns updated row; deciding a non-pending row is an error), `set_approval_execution(id, json)`, `expire_approvals(now_ms) -> u32` (pending past expires_at → expired), `insert_agent_run/update_agent_run_status(id, status, ended?, result_summary?, diffstat?)/recent_agent_runs(n)/get_agent_run(id)`, `insert_blob(bytes) -> (id, sha256)` deduping on sha256, `get_blob(id)`. Tests: migration v3→v4 on populated db; approval state machine incl. double-decide error + expiry; blob dedup.
Commit: `feat(store): migration v4 — approvals, actions, agent_runs, blobs`

### Task 2: rat-policy crate
`ActionKind` enum: `SensorRead, TranscriptParse, WorktreeCreate, WorkbenchFileWrite, Pin, DotfileEditManaged, McpKnownEdit, CommandOutsideWorktree, TerminalInject, MergeBack, LiveRepoWrite, ProjectLocalInstall, McpNewBinary, GlobalInstall, ShellStartupEdit, ConfigOutsideSafe, GitForcePush, SudoAnything`. `risk_tier(kind) -> Tier` exactly per §11 table (R0: SensorRead, TranscriptParse; R1: WorktreeCreate, WorkbenchFileWrite, Pin, DotfileEditManaged, McpKnownEdit; R2: CommandOutsideWorktree, TerminalInject, MergeBack, LiveRepoWrite, ProjectLocalInstall, McpNewBinary; R3: GlobalInstall, ShellStartupEdit, ConfigOutsideSafe, GitForcePush; SudoAnything → `Refused` variant, never proposable). `requires_approval(tier) -> bool` (R2|R3), `requires_slug(tier) -> bool` (R3). NO mutation API (hard invariant §11). Table-driven test covering every variant.
Commit: `feat(policy): table-driven R0–R3 risk tiers (sudo: refused)`

### Task 3: rat-workbench crate — tmux + worktrees
`tmux.rs`: `Tmux { socket_name }` (`new("rato")`, tests use `"rato-test-<pid>"`): `ensure_server()`, `ensure_session(name)`, `new_window(session, name, cwd) -> Target` (string `session:window`), `run_in_window(target, cmd)` (send-keys with literal flag), `capture_tail(target, lines) -> String`, `window_alive(target) -> bool`, `kill_window/kill_server`. All `std::process::Command` shell-outs, never `sh -c` with interpolated strings (args passed as separate argv items).
`worktree.rs`: repo-hash = first 12 hex of sha256(canonical repo path). `create(repo_root, task_id, slug, base) -> Worktree { path, branch }` (`git worktree add -b rato/<slug> <path> <base>`); `diffstat(w) -> String` (`git diff --stat <base>...HEAD` in worktree), `full_diff(w) -> String`, `commits_ahead(w) -> u32`, `remove(w)` (worktree remove --force + keep branch), `is_fast_mergeable(repo_root, branch) -> bool` (`git merge-tree --write-tree` exit 0 + no conflict markers; fall back to `git merge --no-commit --no-ff` dry-run in a TEMP CLONE if merge-tree unsupported — check git version ≥2.38 at runtime), `merge_into_live(repo_root, branch) -> Result<MergeOutcome>` (`git merge --no-ff --no-edit`), `escape_guard(worktree_root, cwd) -> Result<()>` (canonicalize both; cwd must start with root). Env scrub helper for agent spawn: remove `SSH_AUTH_SOCK`, `GIT_DIR`, `GIT_WORK_TREE`.
Integration tests (guard with `which tmux` skip): throwaway repo (tempdir, git init, seed commit) + tmux `-L` unique socket: full create→window→run→capture→remove cycle; escape_guard refuses /tmp; diffstat correctness.
Commit: `feat(workbench): tmux shell-out wrapper + worktree lifecycle with escape guard`

### Task 4: rat-workbench — adapters + TaskRunner
`adapter.rs`: trait per spec. `FakeAgent`: `headless_cmd` = `bash <repo>/tests/fixtures/fakeagent.sh` (CREATE this fixture: reads env `FAKEAGENT_SCRIPT` with lines like `write:file.txt:content`, `commit:msg`, `exit:0`, executes them in cwd; default behavior: write `AGENT_NOTE.md`, git add+commit "fakeagent: scripted change", exit 0). `ClaudeCode`: detect `claude` on PATH; `claude -p "<task>" --output-format json --dangerously-skip-permissions=false`; `Codex`: detect `codex`; `codex exec "<task>"`. `parse_transcript`/`transcript_dirs`: FakeAgent → none; real adapters → stub returning empty (full parsing is M7 terminal work; document).
`runner.rs`: `TaskRunner { store, tmux, clock }`. `start(project_root, project_id, title, adapter, base) -> AgentRun`: insert agent_run (status `running`), create worktree, ensure session/window, run adapter headless_cmd via tmux window wrapped in `cd <worktree> && <cmd>; echo RATO_EXIT=$?` (cwd enforced + exit visible in capture), store tmux_target+worktree+branch on the run. `poll(run_id)`: capture tail, look for `RATO_EXIT=(\d+)` → status done/failed + ended; also failed if window gone w/o marker. `merge_back(run_id) -> Approval`: diffstat+full_diff (diff >32KB → blob, reference id in payload), policy tier = MergeBack (R2), insert approval (expires 60min) with payload {branch, target: repo_root, diffstat, diff_or_blob}. `execute_merge(approval) -> MergeOutcome`: re-check approval approved + not expired; is_fast_mergeable else `NeedsManual`; merge; write execution result onto approval + agent_run status `merged`. `deny` path only updates approval; assert worktree/branch untouched.
Tests: fakeagent fixture e2e in tempdir repo + test socket (start→poll until done→merge_back→approve→execute_merge → live repo HEAD moved, file present; SECOND run with deny → `git rev-parse HEAD` + `git status --porcelain` byte-identical before/after, branch still exists); needs_manual on conflicting branch; oversized diff → blob path.
Commit: `feat(workbench): agent adapters (fakeagent, claude-code, codex) + task runner with R2 merge-back`

### Task 5: daemon + proto + CLI
RPC per spec §4 (workbench.start/runs/tail, approvals.pending/decide). decide: R3 requires `slug` param matching approval's stored slug (last 6 of id, rendered on the card); R2 decide(approve) triggers execute_merge for MergeBack approvals. 60s expiry sweep task. CLI: `rat task start --project <path> --title <t> [--adapter fakeagent|claude-code|codex]`, `rat task list`, `rat task tail <run_id>`, `rat approvals`, `rat approvals decide <id> approve|deny [--note] [--slug]`. TestDaemon integration tests for each method (seed via store; workbench.start against a tempdir repo with fakeagent).
Commit: `feat(daemon,cli): workbench + approvals RPC and CLI`

### Task 6: shell — Workbench + Approvals tabs
Workbench tab: runs table (adapter, title, status chip, started rel-time, diffstat summary), expandable tail (poll 2s via workbench.tail when expanded), "Merge back" hud-btn on done runs → approvals.pending refresh. Approvals tab: pending ApprovalCards — paper card with risk-striped left border (R1 ok-green, R2 warn-amber, R3 danger-red), title, reason, diffstat block (mono), expiry countdown, Approve/Deny buttons; R3 adds a text input "type <slug> to arm Approve" (button disabled until match); audit list (decided/expired) below, collapsed. Avatar: amber `APR` chip in grip when pending>0 (extend existing 2s poll with approvals.pending count; reuse StatusChip with state warn). DTOs in types.ts matching proto.
Verify: npm check/build green.
Commit: `feat(shell): Workbench + Approvals tabs (risk-striped cards, R3 slug arming)`

### Task 7: acceptance + tag (controller-led, live)
Scratch repo `~/rato-scratch-m4` (git init + seed). Via CLI: start fakeagent task → `rat task list` running→done; worktree + branch exist; scratch HEAD unchanged. `rat task tail` shows output.
