# M7 — Polish Implementation Plan

> **For agentic workers:** implement task-by-task. The deterministic core is the acceptance bar for
> default builds: terminal classification, injection ceremony state machine, DotfileEditor
> chokepoint, transcript parsing, read-only dashboard assembly, and graceful avatar fallback.
> Desktop injection/focus and glTF art assets remain feature/runtime-gated and operator-smoked.

**Goal:** full polish layer per `docs/superpowers/specs/2026-06-11-m7-polish-design.md`:
terminal detection/injection, DotfileEditor/MCP audit+revert, transcript ingestion, avatar
personality/render polish, and remaining dashboard tabs.

**Migration note:** the M7 spec says v7, but this working tree already uses v7 for M6
`voice_utterances`. M7 schema additions therefore land as **v8**.

**Conventions:** default build must not require live desktop injection tools or avatar assets;
terminal injection must be approval-ticketed and JIT-rechecked; away mode blocks injection; bracketed
paste and Enter are separate approved actions; every managed config write goes through
DotfileEditor and is reversible.

### Task 1: store v8 terminals + dotfile edits

Migration v8:

- `terminals(id PK, tty, pid, emulator, tmux_target NULL, role, project_id NULL, cmd_hash,
  first_seen, last_seen, meta JSON)`
- `dotfile_edits(id PK, path, kind, before_blob, after_blob, diff, reason, source, risk,
  created, applied INT, reverted_by NULL, meta JSON)`
- indexes on `terminals(tty)`, `terminals(role,last_seen)`, and `dotfile_edits(created)`

Store API:

- upsert/list/get terminal rows
- insert/list/get dotfile edit rows
- mark dotfile edit reverted by another edit id

Tests: v7→v8 preserves voice utterances; terminal upsert refreshes last_seen and role; dotfile edit
round-trips blob ids and links `reverted_by`.

### Task 2: `rat-terminal` detection/classification

New crate with `ProcSource` trait and fake fixtures. Detect adapter process names
(`claude`, `codex`, `aider`, `gemini`, configured extras), resolve TTY, map tmux panes, walk parent
chain for emulator, classify RATO workbench vs foreign/operator/ignored.

Tests: proc/tmux fixtures for RATO workbench, foreign Claude/Codex terminal, ignored remembered
terminal, and no false positive for non-agent processes.

Status 2026-06-16: deterministic classifier is backed by a best-effort real `/proc` source and
`tmux list-panes` mapper. The daemon sensor loop scans every 10 seconds, upserts terminal rows, and
reuses stored operator/ignored command hashes during classification. First-sighting dialogue remains
shell-side.

### Task 3: `rat-inject` ceremony state machine

Pure state machine plus `Injector` trait and fake injector. Path selection order:
tmux, portal/libei, ydotool, xdotool. Real backends are feature/runtime gated.

Tests: no execute without approval, expiry aborts, JIT target mismatch aborts, away blocks even after
approval, paste is bracketed, Enter is separate.

### Task 4: `rat-dotfile` chokepoint

Atomic validating editor for known config surfaces. It snapshots before/after blobs, validates
JSON/JSONC/TOML/YAML and known MCP command paths, applies temp+rename, records a dotfile edit event,
and supports byte-identical revert through a linked edit.

Status 2026-06-16: deterministic core validates JSON/JSONC/TOML/YAML, rejects missing MCP command
paths, applies by same-directory temp+rename, and reverts exact bytes. Daemon RPC/CLI can apply new
managed edits through this chokepoint, store before/after blobs, list audit rows, and perform linked
byte-exact reverts.

Tests: validation rejects malformed configs/missing binaries, no half-apply on failure, revert restores
exact bytes, random edit proptest around atomicity.

### Task 5: transcript parsers

Implement Claude Code and Codex transcript dirs/parsers and ingest parsed turns as
`agent_output` observations. Aider/Gemini stubs remain for M8.

Tests: fixture `.jsonl` transcripts parse expected speaker/timestamp/text rows.

Status 2026-06-16: daemon sensor loop scans recent Claude/Codex agent runs, walks each adapter's
project-local transcript dirs for `.jsonl`, parses them through the adapter, records `agent_output`
observations with run/adapter/path metadata, and dedupes by transcript file key.

### Task 6: daemon/CLI/RPC

Add read/write surfaces:

- `terminals.list`, `terminals.set_role`
- `dotfile_edits.list`, `dotfile_edits.apply`, `dotfile_edits.revert`
- `rat terminals`
- `rat config-edits [apply|revert <id>]`

Wire first-sighting terminal rows and dotfile edit audit rows without enabling real injection by default.

Status 2026-06-16: `terminals.list`, `terminals.set_role`, `dotfile_edits.list`,
`dotfile_edits.apply`, and `dotfile_edits.revert` are wired through daemon/RPC/CLI with tests. Real
scanner storage is wired; first-sighting UI remains.

### Task 7: shell/avatar polish

Complete Metrics, Memory, and Pushback tabs from existing store/RPC surfaces. Add injection approval
card exact-bytes rendering and countdown overlay. Avatar loads glTF/PS2 shader assets when present and
keeps the current placeholder fallback when absent.

Status 2026-06-17: the `Now` tab now includes a terminal first-sighting dialogue surface with
operator/workbench/ignore actions, and `Settings` includes a config-change feed with revert actions
and diff previews.

Verify `npm run check` and `npm run build`.

### Task 8: acceptance

Automated:

- `cargo test --workspace`
- `cargo clippy --workspace --all-targets -- -D warnings`
- shell `npm run check` and `npm run build`

Manual/operator:

- foreign Claude terminal detected and classified
- approved tmux paste-and-enter executes with ceremony
- `.claude` edit auto-applies and one-click revert restores original bytes

## Self-Review

The dangerous part of M7 is not parsing terminals; it is acting in them. The state machine and
approval-ticket boundary are the core safety property. Dotfile editing has a similar shape: one
audited chokepoint with snapshots and revert, never scattered writes.
