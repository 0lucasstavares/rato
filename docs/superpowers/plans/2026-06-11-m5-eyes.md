### Task 1: store migration v5 + PinRepo
Migration v5 (additive v4→v5, user_version 5): `pins(id PK, kind, media, path, created, expires_at NULL,
reason, meta JSON)` per §10; index `pins(expires_at)`. Row structs `Pin`, `NewPin`. Store methods:
`insert_pin(NewPin)->Pin`, `list_pins()->Vec<Pin>` (newest first), `get_pin(id)->Option<Pin>`,
`expire_pins(now_ms)->Vec<Pin>` (returns + DELETES rows whose `expires_at` is non-null and `<= now`; returns
the deleted rows so the caller can unlink their files), `delete_pin(id)`. Tests: migration v4→v5 on populated
db; `expire_pins` deletes auto-pins past expiry, leaves `expires_at IS NULL` (manual) and future ones; returns
the deleted rows. Follow the T1/M4 store patterns exactly.
Commit: `feat(store): migration v5 — pins`

### Task 2: rat-ring crate (encrypted ring buffer)
New crate `crates/rat-ring`. `RingKey`: 32-byte key; `ephemeral()` generates a random key (seed the RNG from
`OsRng`), stored in a buffer that is `mlock`ed if possible (use `region` crate or a tiny libc mlock wrapper;
on failure log + proceed) and `zeroize`d on drop. `seal(key, plaintext, aad)->Vec<u8>` / `open(key, sealed,
aad)->Result<Vec<u8>>` using `chacha20poly1305::XChaCha20Poly1305` (24-byte nonce prepended to ciphertext;
nonce from OsRng). `RingWriter { dir, segment_secs, ttl_secs, clock: Arc<dyn Clock> }`:
`write_segment(media, bytes)->SegmentId` (writes `<dir>/<media>/<created_ms>-<ulid>.seg` sealed with the
ephemeral key, aad = media), `prune()->u32` (deletes segments whose created_ms < now-ttl; uses clock),
`list_segments(media)->Vec<Segment{id,path,created_ms}>`, `read_segment(id, key)->Result<Vec<u8>>`. Tests
(fake clock, tempdir): seal/open round-trip; open with a different key fails (AEAD); rotation — write segments
across simulated time, `prune()` keeps only those within ttl, count bounded; nonce uniqueness across two seals
of identical plaintext (ciphertexts differ).
Commit: `feat(ring): ephemeral-key XChaCha20-Poly1305 ring buffer with clock-driven rotation`

### Task 3: rat-vision crate — capture pipeline (trait core + fakes)
New crate `crates/rat-vision`. PURE-LOGIC + TRAIT CORE, default build pulls NO heavy/system deps.
- `dhash.rs`: `dhash(gray_9x8_or_image)->u64` and `hamming(a,b)->u32`. Accept a simple `&GrayImage`/`&[u8]`
  abstraction so tests use synthetic pixel buffers (use the `image` crate — pure Rust, fine as a default dep).
- `screen.rs`: `trait ScreenSource { fn grab(&mut self) -> ScreenResult; fn health(&self) -> SourceHealth }`
  where `ScreenResult = Frame{rgba, width, height, window_title: Option<String>, captured_ms} | Unavailable`,
  `SourceHealth = Ok | Unavailable(reason)`. `FakeScreenSource` (scripted Vec<ScreenResult>).
- `ocr.rs`: `trait OcrEngine { fn recognize(&self, frame: &Frame) -> Vec<OcrBlock>; fn health(&self)->SourceHealth }`,
  `OcrBlock{ text, bbox }`. `NullOcr` (empty, health Unavailable("not built")), `FakeOcr{ scripted: Vec<Vec<OcrBlock>> }`.
- `pipeline.rs`: `CapturePipeline { source, ocr, dedup_distance:4, last_hash, last_ocr_text }`:
  `tick(clock) -> Option<CaptureOutput>` where it grabs a frame, computes dHash, returns `None` if within
  distance 4 of last kept (dedup), else updates last_hash, runs OCR, diffs OCR text vs `last_ocr_text`
  (line-set diff — only NEW/changed lines), returns `CaptureOutput{ frame_jpeg: Vec<u8>, ocr_delta: String,
  window_title, captured_ms }` (jpeg encode via `image`; q70). Empty ocr_delta is allowed (frame still ringed).
- `autopin.rs`: `fn autopin_reason(ocr_delta:&str)->Option<String>` — regex pre-filter for stack-trace/panic/
  error-dialog patterns (e.g. `panicked at`, `Traceback (most recent call last)`, `error\[E\d+\]`, `Exception`,
  `FAILED`), returns the matched reason or None. Table-driven test.
Tests: dHash dedup (near-identical buffers → skip; distinct → keep); pipeline tick with FakeScreenSource+FakeOcr
produces expected ocr_delta diffs across frames; Unavailable source → tick returns None + health surfaced;
autopin regex table.
Commit: `feat(vision): screen/ocr trait pipeline (dhash dedup, ocr-delta, regex auto-pin) with fakes`

### Task 4: rat-vision real backends (feature-gated, best-effort) + daemon capture loop + pins service
- `crates/rat-vision/src/screen_portal.rs` (`#[cfg(feature = "screencast")]`): `PortalScreenSource` via `ashpd`
  ScreenCast + PipeWire frame grab; persistent restore token in keyring (`rato/screencast-token` — reuse
  rat-brain keys pattern). Optional deps `ashpd`, `pipewire` gated by feature `screencast`.
- `crates/rat-vision/src/ocr_tesseract.rs` (`#[cfg(feature = "ocr")]`): `TesseractOcr` via `leptess` (optional
  dep gated by feature `ocr`). If `leptess` cannot be made to build, gate it out and leave a documented stub +

- daemon: a screen-capture loop (2s tick, only when SensorGate screen sensor enabled AND source health Ok) that
  runs `CapturePipeline::tick`, writes the jpeg to the ring (`RingWriter`), and on non-empty ocr_delta inserts
  an `observations(kind="ocr", content=ocr_delta, meta={window_title})` row (reuse the existing observation
  insert + FTS path from M3) and, if `autopin_reason` matches, creates an auto-pin (Task-4 pin service).
  In the default build (no `screencast`), construct `FakeScreenSource`/`NullOcr` so the loop is inert/`unavailable`.
- pins service (in daemon or a small `pins.rs`): `pin_recent(media, minutes, kind, reason)` — take the ring
  segments for `media` within the last `minutes`, re-encrypt each with the **persistent pin key** (keyring
  `rato/pin-key`, created on first use) into `~/.local/share/rato/pins/<pin-id>/`, insert a `pins` row

  for expiry. RPC (additive): `pins.pin_recent {media, minutes}` (kind=manual), `pins.list`, `pins.unpin {id}`.

Tests: daemon integration — drive the loop with a `FakeScreenSource` (inject via constructor) for a couple of
ticks, assert an `ocr` observation lands and is returned by `rat search`/observations; `pins.pin_recent` over a
ring with seeded segments creates a pin dir + row; `pins.unpin` removes both. `cargo check --features
screencast,ocr` is attempted; if it fails ONLY due to missing system libs (pipewire/tesseract/leptonica),
that's expected — record it; pure-Rust glue errors must be fixed.

#### Task 4 — implementation notes (resolved decisions, ready to build)
Captured 2026-06-11 (T1–T3 done; T4 specced, NOT implemented). Concrete choices the implementer should follow:
- **Paths:** add `rat_core::paths::state_dir()` mirroring `data_dir()` but rooted at `$XDG_STATE_HOME` / `~/.local/state` (`rato/`). Ring → `state_dir()/ring/<media>/`; pins → `data_dir()/pins/<pin-id>/`.
- **Ring construction in the daemon:** one `RingWriter` for `Media::Screen`, `segment_secs = 10`, `ttl_secs = 1200` (20 min), with a single process-lifetime `RingKey::ephemeral()` held in daemon state (already mlock'd + zeroize-on-drop by `RingKey`). This ephemeral key encrypts the live ring; a crash makes old ring garbage (intended).
- **Persistent pin key:** a separate 32-byte key persisted in Secret Service `rato/pin-key`, created on first pin, loaded via the rat-brain `keys.rs` off-runtime-thread keyring pattern. `pin_recent` reads each in-window segment with the **ephemeral** ring key (`ring.read_segment`) and re-`seal`s the plaintext under the **persistent** pin key into the pin dir, so pins survive daemon restarts (unlike the ring). Reuse `rat_ring::seal/open` with a `RingKey` built from the persistent bytes (add a `RingKey::from_bytes([u8;32])` constructor to rat-ring if one isn't present — small additive change).
- **Loop is inert by default:** in the default build construct the pipeline from `FakeScreenSource` (empty script → `Unavailable`) + `NullOcr`; the 2s loop must early-return cheaply when `source.health()` is `Unavailable` (no ring writes, no observations). The real `PortalScreenSource`/`TesseractOcr` only swap in under `--features screencast`/`--features ocr` (T5 wires SensorGate health to report `unavailable`).
- **Test seam:** expose a constructor/`#[cfg(test)]` hook on the daemon ctx (or a free `run_capture_tick(pipeline, ring, key, store, clock)` function the loop also calls) so a test can drive ticks with `FakeScreenSource`+`FakeOcr` deterministically and assert the `ocr` observation + ring segment. Prefer a free function over a private loop body so it's unit-testable without spawning the daemon.
- **Observation insert:** reuse the EXACT existing observation+FTS path (the one M1/M3 use for `kind=shell_cmd`/`agent_output`); do not write a parallel insert. `meta = {"window_title": ...}`; `content = ocr_delta`.
- **Feature gating discipline:** `screencast`→optional `ashpd`+`pipewire`, `ocr`→optional `leptess`, declared `optional = true` + a `[features]` table in `rat-vision/Cargo.toml`; default features empty so `cargo build`/`cargo test --workspace` never pull them. If a real backend can't be wired confidently without uninstallable system libs, ship a `#[cfg(feature=...)]` stub returning `Unavailable` with a `// TODO(operator): real impl needs <lib>` note — the default build + tests are the acceptance bar, the live backend is an operator smoke step.
Commit: `feat(vision,daemon,cli): capture loop + pins (manual pin-recent, persistent pin key) + feature-gated portal/tesseract`
