# M6 — Voice Implementation Plan

> **For agentic workers:** implement task-by-task. The deterministic core is the acceptance bar for
> default builds: backend traits, fake implementations, RAM-only pre-wake ring, intent routing,
> slug-gated voice approvals, daemon/RPC/CLI/shell surfaces, and tests. Live mic/wake/STT/TTS
> backends are feature-gated and may report `unavailable` until host assets and consent exist.

**Goal:** wake-word / VAD / STT / local intents / TTS / voice approval, per
`docs/superpowers/specs/2026-06-11-m6-voice-design.md` and ARCHITECTURE §15.

**Status 2026-06-16:** most deterministic core work already exists in the working tree:
`rat-voice`, store v7 `voice_utterances`, daemon `voice.status`/`voice.utterances`, CLI voice
commands, Settings voice controls, Approvals spoken slug, and avatar MIC pulse. Live hardware
backends and operator smoke remain.

**Conventions:** default build must not pull mic/model/TTS system deps; every time-sensitive path
uses `Clock`; pre-wake audio must stay RAM-only; voice approval must require a visible pending R2-or
lower approval plus the two-word spoken slug; R3 is never voice-approvable. Keep RPC additions
additive under protocol v1.

### Task 1: store v7 voice utterances

Migration v7: `voice_utterances(id PK, ts, lang, text, intent NULL, wake_word, handled INT)`.
Store API:

- `insert_voice_utterance(NewVoiceUtterance) -> VoiceUtterance`
- `recent_voice_utterances(limit) -> Vec<VoiceUtterance>` newest first

Tests: v6→v7 preserves `retention_status`; insert/recent round-trip; bounded limit ordering.

### Task 2: `rat-voice` deterministic core

New crate `rat-voice` with:

- backend traits: `AudioSource`, `WakeDetector`, `Vad`, `SttEngine`, `TtsEngine`
- fake implementations for tests
- `PrewakeRing`: 8 s RAM-only bounded buffer, zeroized on clear/drop, no serialize/write API
- `IntentRouter`: English and Portuguese local grammar for pause/resume, private on/off,
  dashboard, `pin that` / `pina isso`, snooze, mode switch, approval approve/deny slugs,
  and fallback chat
- two-word spoken slug derivation from approval id
- voice approval gate: visible pending approval, risk <= R2, correct spoken slug; reject hidden
  popup, R3, wrong slug, and non-pending approvals

Tests: grammar tables, slug determinism, pre-wake bounds/clear, fake path non-persistence, and
security-critical voice approval matrix.

### Task 3: feature-gated real backend shells

Feature flags:

- `mic`: PipeWire microphone source
- `wake`: openWakeWord + Silero VAD
- `stt`: whisper-rs STT
- `tts`: Piper TTS

Default build exposes Null/Unavailable implementations only. If host libraries/models/assets are
missing, report `unavailable(reason)` through status rather than fabricating capability.

Verification: `cargo check --workspace` default never pulls hardware/model deps; optional feature
checks are best-effort and documented.

### Task 4: daemon voice loop and RPC

Daemon wiring:

- maintain voice backend health/status
- fake-testable loop: pre-wake ring → wake → VAD → STT → insert `voice_utterances` → route intent
- local intent execution for voice approvals and screen-ring `pin that` / `pina isso`
- record voice approval decisions with `decided_via = "voice"` and utterance id
- additive RPCs: `voice.status`, `voice.utterances`

Tests: fake loop records utterances; local pin intent creates a manual screen pin when a recent
segment exists; voice approval path records the voice decision metadata.

### Task 5: CLI

Add:

- `rat voice status`
- `rat voice say "<text>"`
- `rat utterances --limit N`

Default behavior reports unavailable live backends without failing. Integration tests run against an
in-process daemon.

### Task 6: shell/avatar

Shell work:

- Approvals tab renders spoken slug on pending approval cards
- Settings tab shows voice backend status, wake/listen toggles, language smoke controls, recent
  post-wake utterances
- avatar MIC chip follows `voice.status` and pulses on new post-wake utterance

Verify `npm run check` and `npm run build`.

### Task 7: acceptance

Automated:

- `cargo test --workspace`
- `cargo clippy --workspace --all-targets -- -D warnings`
- `cargo check -p rat-daemon --features mic,wake,stt,tts` if optional deps are present; otherwise
  record the host dependency blocker
- shell `npm run check` and `npm run build`

Manual/operator:

- live smoke with mic/wake/STT/TTS features and model assets installed
- both English and Portuguese wake and command
- Piper speaks both configured voices
- pre-wake audio remains non-persistent by code audit plus fs-watch test evidence

## Self-Review

M6 is security-sensitive because voice can approve work. The approval gate stays separate from the
general intent parser, requires UI visibility and a spoken slug, and refuses R3 regardless of slug.
The pre-wake ring is structural, not policy-only: it has no disk write path. Hardware backends are
replaceable seams so tests remain deterministic and default builds remain host-independent.
